# 基于小程序的门禁预约管理平台 - 系统设计文档

## 一、系统概述

本系统是一个基于微信小程序的宿舍门禁预约管理平台，主要用于管理学生宿舍楼的门禁预约，实现预约申请、审批、门禁控制等功能。学生可以通过小程序预约宿舍楼门禁的使用时间，管理员负责审核预约申请，确保宿舍门禁的有序使用。

## 二、角色设计

### 2.1 普通用户（学生）
- **权限**：申请宿舍门禁预约、查看个人预约记录、取消预约
- **功能**：
  - 查看可预约时间段
  - 提交预约申请（选择宿舍楼、日期、时间段）
  - 查看预约状态（待审核/已通过/已拒绝/已使用）
  - 取消未使用的预约
  - 查看历史预约记录
  - **常用门禁收藏**：收藏常用宿舍楼门禁位置，快速预约
  - **预约日历视图**：以日历形式查看个人预约记录，直观展示预约分布
  - **个人使用统计**：查看个人预约次数、使用频率等统计数据
  - **预约提醒设置**：设置预约前提醒时间（如提前30分钟提醒）
  - **访客预约申请**：为访客预约进出时间，填写访客姓名/电话/来访事由（提交后走管理员审核）
  - **晚归/临时外出报备**：学生可提交晚归/临时外出报备，便于宿舍管理（提交后走管理员审核）
  - **出入记录查询**：查看个人门禁使用记录（时间、宿舍楼门禁、状态）
  - **宿舍报修/反馈**：提交门禁相关问题反馈（如刷不开、设备故障），支持图片与处理进度查看
  - **紧急联系人**：维护紧急联系人信息，用于突发情况联系（可选）

### 2.2 管理员
- **权限**：审核预约申请、管理门禁设备、查看统计数据
- **功能**：
  - 审核预约申请（通过/拒绝）
  - **批量审核功能**：支持批量通过或拒绝多个预约申请，提高审核效率
  - 管理门禁设备信息（添加/编辑/删除）
  - **设备维护管理**：记录设备故障、维护记录，设置设备维护状态（正常/维护中/故障）
  - 设置可预约时间段
  - **预约规则设置**：设置预约限制规则（如每人每天最多预约次数、提前预约时间限制等）
  - 查看预约统计报表
  - **预约数据导出**：支持将预约记录导出为Excel文件，便于数据分析
  - 管理用户信息
  - **用户黑名单管理**：对违规用户进行黑名单管理，限制其预约权限
  - 查看所有预约记录

### 2.3 超级管理员（系统管理员）
- **权限**：系统配置、用户管理、角色权限管理
- **功能**：
  - 用户账号管理
  - 角色权限分配
  - 操作日志查看

## 三、核心功能模块

### 3.1 用户管理模块
- 用户注册/登录
- 个人信息管理
- 角色权限管理

### 3.2 门禁设备管理模块
- 宿舍楼门禁设备信息维护
- 设备状态监控
- 宿舍楼位置管理

### 3.3 预约管理模块
- **预约申请**：
  - 选择宿舍楼门禁位置
  - 选择日期和时间段
  - 填写预约事由（如：回宿舍、访客等）
  - 提交申请
  
- **预约审核**：
  - 待审核列表
  - 审核操作（通过/拒绝）
  - 审核意见填写
  
- **预约查询**：
  - 个人预约记录
  - 预约状态查询
  - 预约统计

### 3.4 门禁使用模块
- 预约使用确认（用户点击确认使用）
- 使用记录保存
- 预约状态更新

### 3.11 访客预约模块（学生端）
- 学生发起访客预约申请（访客信息、来访时间、事由）
- 查看访客预约状态与历史记录
- 取消未开始的访客预约

### 3.12 晚归/外出报备模块（学生端）
- 提交晚归/外出报备（预计时间、原因说明）
- 查看报备审核状态与历史记录
- 支持撤销未审核的报备

### 3.13 出入记录模块（学生端）
- 查询个人出入记录（按日期/宿舍楼筛选）
- 异常记录标记（如：超时/未在预约时间段内使用）

### 3.14 报修与反馈模块（学生端）
- 提交门禁相关报修/反馈（问题类型、描述、图片）
- 查看处理进度与管理员回复

### 3.7 门禁收藏模块（学生端）
- 收藏常用宿舍楼门禁位置
- 快速预约收藏的门禁
- 管理收藏列表（添加/删除收藏）

### 3.8 预约规则管理模块（管理员端）
- 设置每人每天最大预约次数
- 设置提前预约时间限制（如至少提前1小时）
- 设置预约有效期规则
- 设置特殊时间段预约规则

### 3.9 设备维护管理模块（管理员端）
- 设备故障报修记录
- 设备维护状态管理（正常/维护中/故障）
- 维护记录查询
- 维护提醒功能

### 3.10 黑名单管理模块（管理员端）
- 添加/移除黑名单用户
- 查看黑名单列表
- 设置黑名单期限
- 黑名单用户预约限制

### 3.5 消息通知模块
- 预约审核结果通知
- **预约提醒通知**：预约时间前自动提醒用户（根据用户设置的提醒时间）
- 系统公告
- 设备维护通知

### 3.6 数据统计模块
- 预约数据统计
- 门禁使用统计
- **个人使用统计**（学生端）：个人预约次数、使用频率、常用门禁等
- **预约数据导出**（管理员端）：支持Excel格式导出预约记录

## 四、业务流程设计

### 4.1 预约申请流程
```
学生登录 → 选择宿舍楼门禁位置 → 选择日期和时间段 → 填写预约信息 → 提交申请 → 等待审核
```

### 4.2 预约审核流程
```
管理员登录 → 查看待审核列表 → 查看预约详情 → 审核（通过/拒绝）→ 发送通知
```

### 4.3 门禁使用流程
```
学生到达宿舍楼门禁 → 打开小程序 → 查看预约信息 → 点击确认使用 → 记录使用时间 → 更新预约状态
```

## 六、技术架构

### 6.1 前端技术栈
- **框架**：Vue.js + uni-app
- **应用端**：微信小程序（基于uni-app开发，用户和管理员均通过小程序操作）

### 6.2 后端技术栈
- **框架**：Spring Boot
- **数据库**：MySQL 8.0
- **ORM框架**：MyBatiys
- **安全框架**：JWT Token

## 七、小程序页面结构设计

### 7.1 用户端页面（学生）
1. **首页（index）**：展示可预约宿舍楼门禁列表、快速预约入口、公告通知、收藏门禁快捷入口
2. **预约页面（reservation）**：选择宿舍楼门禁、日期、时间段、填写预约信息、从收藏列表快速选择
3. **我的预约（myReservations）**：预约列表（待审核/已通过/已拒绝/已使用）、状态筛选、日历视图切换
4. **预约详情（reservationDetail）**：查看预约详细信息、取消预约、查看审核意见
5. **门禁使用（use）**：显示预约信息、确认使用按钮、使用记录
6. **预约日历（calendar）**：日历视图展示个人预约记录，点击日期查看当日预约
7. **收藏管理（favorites）**：管理收藏的宿舍楼门禁列表，快速预约收藏的门禁
8. **个人统计（statistics）**：个人预约次数、使用频率、常用宿舍楼门禁等统计数据
9. **访客预约（visitorReservation）**：提交访客预约、查看审核状态、取消预约
10. **晚归/外出报备（report）**：提交报备、查看审核状态、撤销报备
11. **出入记录（accessRecords）**：查看个人出入记录与异常记录
12. **报修与反馈（feedback）**：提交报修/反馈、查看处理进度与回复
13. **紧急联系人（emergencyContact）**：维护紧急联系人信息
14. **个人中心（profile）**：个人信息、提醒设置、收藏管理入口、退出登录

### 7.2 管理端页面（管理员/超级管理员）
1. **管理首页（admin/index）**：待审核数量、今日预约统计、快捷入口、设备状态概览
2. **审核管理（admin/audit）**：待审核列表、预约详情、单个/批量审核操作（通过/拒绝）
3. **设备管理（admin/device）**：设备列表、添加/编辑/删除设备、设备状态管理、维护记录
4. **设备维护（admin/maintenance）**：设备故障报修、维护记录管理、维护状态更新
5. **预约查询（admin/reservations）**：所有预约记录查询、筛选、详情查看、导出Excel
6. **数据统计（admin/statistics）**：预约统计、使用统计、用户活跃度、数据图表展示
7. **预约规则（admin/rules）**：设置预约限制规则、时间段规则、有效期规则
8. **用户管理（admin/users）**：用户列表、用户信息查看、角色管理（仅超级管理员）、黑名单管理
9. **黑名单管理（admin/blacklist）**：黑名单用户列表、添加/移除黑名单、设置期限
10. **系统设置（admin/settings）**：系统参数配置、操作日志（仅超级管理员）

### 7.3 公共页面
1. **登录页（login）**：用户登录、角色识别
2. **消息通知（notifications）**：系统通知、审核结果通知

## 八、关键功能点

### 8.1 预约冲突检测
- 同一时间段同一宿舍楼门禁只能被一人预约
- 学生不能重复预约同一时间段

### 8.2 预约时效性
- 预约需提前一定时间（如1小时）
- 过期预约自动失效
- 使用后预约状态更新

### 8.3 审核机制
- 预约需管理员审核通过
- 紧急预约可设置快速通道（可选）

### 8.4 使用确认机制
- 学生到达宿舍楼门禁后，在小程序中点击确认使用
- 系统记录使用时间，更新预约状态为"已使用"
- 仅已通过审核且在预约时间段内的预约可以确认使用

### 8.5 预约规则限制
- 每个学生每天最多预约次数限制（管理员可配置，如每天最多3次）
- 提前预约时间限制（如至少提前1小时）
- 黑名单学生无法提交预约申请
- 维护中的宿舍楼门禁设备不可预约
- 可设置宿舍楼门禁的开放时间段（如：工作日8:00-22:00，周末9:00-21:00）

### 8.6 批量操作机制
- 管理员可批量选择多个预约进行审核
- 批量操作前需确认操作内容
- 批量操作记录操作日志

### 8.7 数据导出功能
- 支持按时间范围、状态、宿舍楼门禁位置等条件筛选导出
- 导出格式为Excel，包含预约基本信息（学生姓名、学号、宿舍楼、预约时间等）
- 导出文件自动命名（包含导出时间）

### 8.8 访客预约与报备规则（学生端）
- 访客预约需填写访客真实信息（姓名、手机号），并选择来访时间段
- 访客预约与晚归/外出报备默认需要管理员审核（可配置）
- 同一学生同一时间段的访客预约不可重复提交

### 8.9 反馈闭环机制
- 学生提交反馈后生成工单编号，支持状态流转（待处理/处理中/已完成）
- 管理员可回复处理结果，学生可查看处理记录
 
**技术选型**：
   - 前端：Vue.js + uni-app
   - 后端：Spring Boot
   - 数据库：MySQL
   - 开发工具：IDEA
